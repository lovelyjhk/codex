# 릴리스 노트 작성 Skill

## 목적
개발자용 git 커밋 로그를 일반 사용자가 읽을 수 있는 릴리스 노트로 바꾼다.

## 입력
- git 커밋 로그 (텍스트) 또는 `sample-git-log.txt` 파일

## 절차
1. 커밋을 기능 추가 / 버그 수정 / 개선 / 기타로 분류한다.
2. 각 항목을 개발 용어가 아닌 사용자 관점 문장으로 바꾼다.
3. 사용자가 체감하는 변화가 없는 내부 커밋(리팩토링 등)은 "내부 개선"으로 묶는다.
4. 버전 번호와 날짜를 맨 위에 적는다.
5. 커밋 로그에 없는 내용은 추측하지 않는다.

## 출력 형식
- 버전·날짜 헤더
- 새로운 기능 목록
- 버그 수정 목록
- 개선 사항 목록
